Install complete system on blank machine. No external dependencies. Fork everything. Control the stack.


### Right Tool for Right Job

**Rust**: Kernel, drivers, system services (memory safety, performance)  
**Nix**: Package definitions, builds (declarative, reproducible)  
**Clojure**: Applications, scripting (productive, functional)

No dogma. Specialized tools for specialized purposes.

## Technical Architecture

### Kernel Layer (Rust)

**Redox OS** - Rust microkernel
- Memory safety by design
- Microkernel architecture (drivers in userspace)
- x86_64 primary, ARM/RISC-V future
- No C legacy, pure Rust

**Why Redox**:
- Memory safety without garbage collection
- Isolation (crashed driver doesn't kill kernel)
- Modern tooling
- Boot-from-scratch philosophy alignment

### Package Layer (Nix + Alpine)

**Nix** - Declarative, reproducible builds
- Better for packages than Clojure (specialized tool)
- Rollback capability
- Multi-version support
- Rust bindings needed (contribution opportunity)

**Alpine apk** - Binary packages
- musl libc (clean, minimal)
- Small footprint
- Quick installs
- Complements Nix source builds

### Init Layer (s6)

**s6** - Minimal supervision
- Not systemd
- Simple, reliable
- Fast boot
- Redox-compatible philosophy

### Application Layer (Clojure)

**Babashka** - Fast Clojure scripting
- Build scripts (grainbarrel)
- System utilities (graintime, grainbranch)
- Deployment automation (grainflow)

**Clojure** - Full applications
- Business logic
- User interfaces (HumbleUI)
- Data processing

### Multi-Chain Layer

**ICP**: `clojure-icp`, canisters for backend services  
**Hedera**: Enterprise payments, B2B integration  
**Solana L2**: Micropayments, commission payouts  
**Urbit**: Personal server, identity  
**Nostr**: Social, identity backup  

All forked to grain6pbc. Full control. Boot-from-scratch compatible.

## Android Strategy

### Requirements
Must support:
- Tesla app (remote key, car control)
- Spotify (music streaming)
- YouTube Music (music)
- Google Play ecosystem

### Options

**1. GrapheneOS** (Primary Choice)
- Sandboxed Google Play Services
- Privacy-focused
- Regular Android app compatibility
- Most likely to work with required apps

**2. PostmarketOS** (Secondary)
- Linux-based (not Android)
- Waydroid for Android apps
- More control, less compatibility
- Risk: Tesla app may not work

**3. Standard Android** (Fallback)
- Guaranteed compatibility
- Heavy customization possible
- Less sovereignty
- Pragmatic if alternatives fail

**Decision**: Test GrapheneOS first. Tesla app compatibility is critical.

### Format
```
YYYY-MM-DD--HHMM-TZ--moon-NAKSHATRA--asc-SIGN##-sun-##h--TEAMNAME
```

### Components
- **Date**: Holocene calendar (12025 = 2025 + 10000)
- **Time**: 24-hour, 3-char timezone
- **Moon**: Sidereal nakshatra (27 total, fixed-width padding to 12 chars)
- **Ascendant**: Tropical sign (4 chars) + degree (2 digits)
- **Sun**: Tropical house (2 digits + 'h')
- **Team**: 14 team names, fixed-width padding

### Length
- **graintime**: 76 characters exactly

- **graincard**: Centered in 80-char width (2-char margins)
